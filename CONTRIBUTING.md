# Contributing

Thanks for your interest in contributing to cel-ts.

## Quick Guide

- Install dependencies by running `pnpm install`.
- Install `protoc` if you plan to run conformance tests.
- Run `pnpm typecheck`, `pnpm test`, `pnpm lint`, and `pnpm conformance` before opening a PR.
- Keep changes focused and include tests when behaviour changes.
- Follow the existing code style and conventions.

## Pull Requests

- Describe the problem and the approach.
- Link relevant issues if available.
- Small, reviewable PRs are preferred.

## Reporting Issues

- Include a minimal reproduction where possible.
- Mention your runtime (Node/Bun), OS, and package manager.
